##########建立虛擬環境##########
#1 :
之前試寫版是用Anaconda的環境下所開發,
最後打包成exe時發現檔案肥大(24Xmb),
經查詢後發現打包時會將環境中所安裝的套件也一併打包,
且用Anaconda的環境進行打包時很容易失敗,
故這次使用Python乾淨環境, 針對此專案建立個nh_downloader的虛擬環境,
僅安裝這個project所會用到的套件

----------

#2 :
在cmd中使用指令, 欲在將虛擬環境建立在專案的資料夾中
cd "C:\Lei Project\NH_Downloader"
python -m venv nh_downloader

發現沒有回應, 經查找除錯結果是:
安裝的 Python 版本是 py 啟動器
Windows 預設會裝 py（Python Launcher），所以 python 指令可能不存在

解法 : 要改用py而不是python

修正後使用, 在資料夾中可看到建立的虛擬環境資料夾
cd "C:\Lei Project\NH_Downloader"
py -m venv nh_downloader

----------

#3 :
啟用虛擬環境, cmd指令
nh_downloader\Scripts\activate

畫面看到
(nh_downloader) C:\Lei Project\NH_Downloader>

----------

#4 :
在虛擬環境中安裝所需要的套件
pip install requests beautifulsoup4 Pillow

----------

#5 :
讓VS Code使用這個虛擬環境來進行編譯,
右下角會有一個顯示 Python x.x.x 的欄位(我這裡是顯示3.13.6),
點擊=>上方的"Select Interpreter"
=>選擇.\nh_downloader\Scripts\python.exe

##########測試版試跑##########
#1 :
遇到錯誤

PS C:\Lei Project\NH_Downloader> & "C:/Lei Project/NH_Downloader/nh_downloader/Scripts/Activate.ps1"
& : 因為這個系統上已停用指令碼執行，所以無法載入 C:\Lei Project\NH_Downloader\nh_downloade
+   ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    + CategoryInfo          : SecurityError: (:) [], PSSecurityException
    + FullyQualifiedErrorId : UnauthorizedAccess

查詢原因
是PowerShell 的執行政策（Execution Policy）是微軟為了防止惡意腳本執行而設計的安全機制。

解法 : 開啟新的PowerShell,輸入
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

##########用Selenium來進行爬蟲##########
#1 :
安裝Selenium
pip install selenium pillow beautifulsoup4 lxml

這裡我是使用Chrome瀏覽器去進行操作
查看Chrome瀏覽器版本的方法:

chrome://version/

查詢到我現在的瀏覽器版本為139.0.7258.67 (正式版本) (64 位元)

#2 :
下載ChromeDriver

https://googlechromelabs.github.io/chrome-for-testing/#stable

根據前面查到的瀏覽器版本下載, 將ChromeDriver置於專案同資料夾中

#3 :
首次試跑失敗

driver = webdriver.Chrome(executable_path=chromedriver_path, options=chrome_options)   
TypeError: WebDriver.__init__() got an unexpected keyword argument 'executable_path'

查找原因 :
是因為新版 selenium（4.x 以上）已經把 executable_path 改成用 Service 物件來指定了
不能直接當參數傳給 webdriver.Chrome(), 舊版寫法不能用了

ChatGPT提供的修正:
from selenium.webdriver.chrome.service import Service

chromedriver_path = r"C:\Lei Project\NH_Downloader\chromedriver-win64\chromedriver.exe"
service = Service(executable_path=chromedriver_path)
driver = webdriver.Chrome(service=service, options=chrome_options)

#4 :
經過多次測試後依然連線失敗, 使用非 headless 模式, 觀看實際跳出瀏覽器的畫面得到連線失敗的原因

網站多了Cloudflare的驗證, 沒有勾選完成驗證的話會連線失敗

這個問題待解
